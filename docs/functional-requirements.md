# 機能要件

## 対象ユーザー

借金を抱えており、現状の把握と返済計画の改善を求めているユーザー。
スマートフォンで銀行・クレカの明細を確認する習慣がある。
借金していることを直視できない。

---

## データ入力の設計哲学

**「月1回 + 気が向いたとき」で動く設計**

連携リスクを避けながら入力の摩擦を最小化するため、以下の2軸で設計する。

| タイミング | 方法 | 内容 |
|-----------|------|------|
| 月1回 | CSV / スクショ | 銀行・クレカの月次明細をまとめて取り込む |
| 気が向いたとき | 手入力 | アプリ上で支出・収入をその場で登録 |

完全自動化は目指さない。「借金を直視できない人」がアプリを開く心理的ハードルを下げることを最優先とする。

### 3. データ取り込み（スクショ・CSV）

取り込み方法はスクリーンショットと CSV の2種類に対応する。どちらも確認・修正してから確定する流れは共通。

#### 3-A. スクリーンショット取り込み

| # | 機能 | 詳細 |
|---|------|------|
| 3-1 | 画像アップロード | 銀行・クレカ・給与明細のスクショをアップロード |
| 3-2 | ソース種別の選択 | アップロード時に bank / credit_card / salary / other を指定 |
| 3-3 | Gemini による自動解析 | Gemini Vision API が画像から金額・日付・内容を読み取り構造化 |
| 3-4 | 解析結果の確認・修正 | 読み取り結果を一覧表示し、ユーザーが誤りを修正してから確定 |
| 3-5 | 処理状態の表示 | pending / processing / completed / failed の状態を表示 |

#### 3-B. CSV 取り込み

| # | 機能 | 詳細 |
|---|------|------|
| 3-6 | CSV アップロード | 銀行・クレカの明細 CSV をアップロード（複数ファイル可） |
| 3-7 | ソース種別の選択 | アップロード時に bank / credit_card / salary / other を指定 |
| 3-8 | カラムマッピング | CSV のヘッダーをアプリの項目に対応付け。銀行ごとの差異を吸収 |
| 3-9 | プレビュー・確認 | 取り込み予定データを一覧表示し、不要な行の除外・修正をしてから確定 |
| 3-10 | 重複検出 | 同一日付・同一金額・同一摘要のデータが既に存在する場合は警告を表示 |

**対応する CSV 形式（MVP フェーズ）**

| 金融機関 | 対応 |
|---------|------|
| 三菱UFJ銀行 | ✅ |
| 楽天カード | ✅ |
| その他 | カラムマッピングで手動対応 |

## ユーザーストーリー

```
As a ユーザー,
I want to スクリーンショットをアップロードするだけで今月の支出を自動で記録したい,
So that 手入力の手間なく現状を把握できる。

As a ユーザー,
I want to 銀行・クレカの明細 CSV をそのままアップロードしたい,
So that スマホでスクショが撮れない場合でも PC からまとめて取り込める。

As a ユーザー,
I want to 毎月どの支出が無駄かを AI に教えてもらいたい,
So that 何を削ればいいかを自分で考えずに行動変容できる。

As a ユーザー,
I want to 不要支出を削った場合に借金返済が何ヶ月早まるかを見たい,
So that 節約のモチベーションを維持できる。
```

## Post-MVP：ゲーミフィケーション構想

返済を「義務」ではなく「ゲーム」として楽しめる仕組みを導入し、継続利用と行動変容を促す。

### アイデア一覧

| 機能 | 概要 |
|------|------|
| 返済ストリーク | 毎月の返済を続けると連続記録が伸びる。途切れると0リセット |
| 借金撃破バッジ | 1件の借金を完済するとバッジを獲得。コレクション要素 |
| 節約チャレンジ | 「今月サブスクを1つ解約する」などのミッションをクリアするとポイント付与 |
| 返済レベル | 累計返済額に応じてレベルが上がる。レベルに応じてキャラや称号が変わる |
| 削減シミュレーター | 「もし毎月1万円多く返したら」をビジュアルで体験できる |
| 月次レポートカード | 先月との比較で「支出-5%」「返済負担率改善」などを通知表形式で表示 |

### 設計の方針

- ネガティブ（借金がある）をポジティブ（ゲームをクリアしている）に変換する体験を作る
- 数字が「減る」ではなく「勝っている」と感じられるUXにする
- カエセルくんのキャラクターの成長と連動させ、世界観を統一する
- 過度なギャンブル性・依存性を避け、健全な行動変容に特化する
