# 非機能要件

## 1. パフォーマンス

| 項目 | 目標値 | 備考 |
|------|--------|------|
| API レスポンスタイム | 2秒以内 | Gemini API 呼び出しを除く |
| Gemini 画像解析 | 30秒以内 | 非同期処理でポーリング対応 |
| ページ初期表示 | 3秒以内 | Vercel CDN + Next.js SSR |
| ダッシュボード表示 | 1秒以内 | キャッシュ活用 |

---

## 2. セキュリティ

| 項目 | 対応方針 |
|------|---------|
| 通信 | HTTPS 必須（Vercel / Railway とも標準対応） |
| 認証 | Supabase Auth（JWT）。トークンの有効期限は1時間 |
| データアクセス制御 | Supabase Row Level Security（RLS）を全テーブルに設定。`user_id = auth.uid()` のみアクセス可 |
| 画像ストレージ | Supabase Storage のバケットを private に設定。署名付き URL でのみアクセス可 |
| 環境変数 | API キー類はすべて環境変数で管理。コードに直書きしない |
| Gemini API | バックエンド（FastAPI）からのみ呼び出す。フロントから直接 API キーを使用しない |

---

## 3. 可用性・信頼性

| 項目 | 目標値 | 備考 |
|------|--------|------|
| 稼働率 | 99%以上 | Vercel / Railway / Supabase の SLA に依存 |
| Gemini 解析失敗時 | エラーメッセージ表示 + 手入力で代替可能 | フォールバック設計 |
| データ損失 | ゼロ | Supabase PostgreSQL の自動バックアップ |

---

## 4. 使いやすさ（UX）

| 項目 | 方針 |
|------|------|
| デザイン方針 | モバイルファーストで設計。スマホで完結できる UI |
| 対応ブラウザ | Chrome（iOS / Android）、Safari（iOS）を優先対応 |
| スクショアップロード | カメラロールから直接選択できる。複数枚の同時アップロード可 |
| エラー表示 | ユーザーが次にすべきアクションを明示するエラーメッセージ |
| ローディング表示 | Gemini 処理中はプログレス表示でユーザーに状態を伝える |

---

## 5. 保守性・拡張性

| 項目 | 方針 |
|------|------|
| コード管理 | GitHub でバージョン管理。main ブランチへの直プッシュ禁止（PR マージ） |
| 環境分離 | development / production の2環境を用意 |
| スケールアップパス | Railway → AWS ECS、Supabase → AWS RDS への移行を前提とした設計 |
| API 設計 | RESTful。将来的なモバイルアプリ対応も見据えた汎用設計 |

## 7. API コスト管理方針

無料公開するとユーザー増加に伴いAPI費用が線形に増加するため、以下の方針でコストを制御する。

### ユーザーあたりの月次API呼び出し想定

| 処理 | API | 頻度 | 単価目安 |
|------|-----|------|---------|
| スクショ解析 | Gemini Vision API | 月2〜4回 | 数円/回 |
| 月次分析・削減提案生成 | Claude Sonnet 4.6 | 月1回 | 数円〜10円/回 |

→ **1ユーザーあたり月10〜30円程度**の試算。100人で月1,000〜3,000円。

### レート制限の設計

以下の上限をSupabaseのテーブルで使用回数を管理して実装する：

| 機能 | 上限（月） |
|------|-----------|
| スクショ解析 | 5回 |
| CSV取り込み | 無制限（AI不使用のため） |
| 月次分析レポート生成 | 1回 |

### フェーズ別の公開方針

| フェーズ | 方針 |
|---------|------|
| MVP | 自分のみ使用。コストと品質を検証する |
| アーリーアクセス | 招待制で少人数に開放。アフィリエイト収益でAPI費用を賄えるか検証 |
| 一般公開 | アフィリエイト収益が安定したタイミングで制限を緩和 |

### コスト上限アラート

Railway / Anthropic / Google の各コンソールで月額上限アラートを設定し、想定外の費用増加を検知する。目安：月$10を超えたら通知。
