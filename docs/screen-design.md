# 画面設計

## 画面一覧

| # | 画面名 | パス | 認証 |
|---|--------|------|------|
| 1 | ログイン | /login | 不要 |
| 2 | 新規登録 | /register | 不要 |
| 3 | ダッシュボード | / | 必要 |
| 4 | 借金一覧 | /debts | 必要 |
| 5 | 借金登録・編集 | /debts/new, /debts/[id]/edit | 必要 |
| 6 | データ取り込み | /upload | 必要 |
| 7 | 支出一覧 | /transactions | 必要 |
| 8 | 収入一覧 | /incomes | 必要 |
| 9 | 月次分析 | /analysis/[year_month] | 必要 |

---

## 画面遷移図

```
[ログイン] ─── 認証成功 ──→ [ダッシュボード]
[新規登録] ─── 登録成功 ──→ [ダッシュボード]

[ダッシュボード]
  ├──→ [借金一覧]
  │       ├──→ [借金登録]
  │       └──→ [借金編集]
  ├──→ [データ取り込み]
  │       └──→ 解析完了 ──→ [支出一覧] or [収入一覧]
  ├──→ [支出一覧]
  ├──→ [収入一覧]
  └──→ [月次分析]
```

## PC 版レイアウト

### レスポンシブ方針

| ブレークポイント | レイアウト |
|----------------|-----------|
| `< 768px`（SP） | ボトムナビ + 1カラム |
| `≥ 768px`（PC） | サイドバーナビ + メインコンテンツ |

PC 版は SP 版のコンテンツをそのまま流用しつつ、サイドバーナビと横並びの多カラムレイアウトで情報密度を高める。
